# 英语口语闯关宇宙 — 产品设计规格

**日期：** 2026-07-18  
**状态：** 待用户审阅  
**产品代号：** English Speaking Adventure（口语冒险家）  
**目标平台：** Windows 桌面应用  
**技术路线：** Electron + React + TypeScript + Vite（方案 A）

---

## 1. 背景与目标

### 1.1 要解决的问题

新手学雅思/托福口语时常见障碍：

1. **太难** — 一上来考试题，直接放弃  
2. **太枯燥** — 像写作业，没有即时反馈与成就感  
3. **没进步感** — 做了很多练习，不知道自己变强了  

### 1.2 产品定位

不是冷冰冰的「刷题软件」，而是面向**完全小白**的 **英语口语闯关成长产品**：

- 前台：地图、副本、荣誉/复仇、动效与音效  
- 后台：从幼儿园级到雅思/托福的分级题库与自适应难度  

### 1.3 成功标准（第一版「完整核心版」）

用户在第一次使用的 10 分钟内能：

1. 完成引导定级并进入合适地图  
2. 至少体验 3 次「答对 → 庆祝反馈」  
3. 完成一次「今日副本」或等价短练习  
4. 看到错题进入「待翻盘」，答对题出现在「荣誉墙」  

长期使用时能：

- 通过冒险地图从低阶地图推进到考场级地图  
- 被随机弹窗以不烦人的节奏打断并完成短题  
- 在荣誉墙/复仇墙清晰看到自己的强弱分布  

### 1.4 明确非目标（第一版不做）

- 账号体系与云同步（预留接口，第一版纯本地）  
- 付费订阅 / 内购（预留扩展点，第一版免费完整可用）  
- 真人外教 / 服务端 AI 深度评分（第一版：本地规则 + 系统语音识别）  
- iOS / Android 移动端  
- 完整官方雅思/托福真题版权内容（使用自研分级示例题库，风格对标考试任务类型）

## 2. 北极星设计原则

1. **永远让用户觉得自己会一点** — 难度自适应，整体略偏容易  
2. **每约 30 秒至少一次明确反馈** — 对/错/进度/连击  
3. **开口被奖励；错误转化为「复仇剧情」** — 不羞辱，给翻盘路径  
4. **考试目标藏在地图后面** — 前台是玩与成长，后台对齐能力分级  
5. **动效强化情绪，且可关** — 默认「完整商业化」档  
6. **弹窗只出 20～40 秒能完成的短题** — 保护「愿意开通知」的习惯

## 3. 用户与使用场景

### 3.1 核心用户

- 英语口语基础弱、想最终冲雅思/托福的成人或学生  
- 能接受「游戏化」与轻度搞怪反馈  
- 主要在 Windows 电脑前学习/摸鱼间隙练习  

### 3.2 关键场景

| 场景 | 行为 |
|------|------|
| 首次打开 | 引导 → 5 题热身定级 → 落入地图 → 送首枚勋章 |
| 每日主动学 | 打开「今日副本」，8～12 分钟闭环 |
| 碎片时间 | 托盘常驻，通知 + 置顶小窗随机短题 |
| 薄弱突破 | 在「待翻盘 / 复仇墙」主动点题重练 |
| 回顾成就 | 荣誉墙、成就页、周报看进步 |

## 4. 产品体验设计

### 4.1 世界观与地图（前台）

用户角色：**口语冒险家**。从低阶地图走向考场地图。

| 地图显示名 | 能力大致对应 | 题型重心 |
|------------|--------------|----------|
| 咿呀村 | 幼儿园～小学低 | 听音选图、超短词、跟读单词 |
| 单词小镇 | 小学～初中词句 | 选词、填空、跟读短句 |
| 对话山谷 | 日常短对话 | 选下一句、情景跟读 |
| 校园港 | 校园/生活场景 | 短说 15～30 秒、情景问答 |
| 考场前线 | 四六级口语感 | 稍长回答、简单话题卡 |
| 托福城 | TOEFL 口语任务感 | 独立口语、听后说（简化版） |
| 雅思塔 | IELTS 口语 1～3 | Part1 短答 → Part2 长谈 → Part3 讨论感 |

**原则：** 用户看到的是地图与故事名；系统内部使用 `levelTier`（0～6）与题目标签对齐。

### 4.2 首次引导（Onboarding）

1. 约 30 秒欢迎：「今天不考试，只玩 3 个小游戏」  
2. **极简定级**：5 道从超简单逐渐变难的热身题（非长测）  
3. 根据正确率与最高答对难度，**宁可定低一点**，保证连赢体验  
4. 落入对应地图，发放 **第一枚勋章** + 礼花  
5. 引导打开「今日副本」或完成 3 题快速胜利  

要求：第一天内至少 **3 次**「答对 → 礼花/音效/升级感」。

### 4.3 核心成长回路

```
抽到「刚好会一点」的题
  → 混合互动（点选 / 跟读 / 短说）
  → 答对：礼花 + 音效 + 经验 + 连击
  → 答错：巴掌动效（可调强度）+ 正确答案 + 跟读/再试
  → 错题进入「待翻盘」，并提高被弹窗抽中的权重
  → 复仇成功：翻盘礼花 + 从复仇墙迁向荣誉墙
  → 连对 / 升级 / 点亮地图 → 继续动机
```

### 4.4 答题反馈（商业化动效）

| 结果 | 视觉 | 听觉 | 文案调性 |
|------|------|------|----------|
| 答对 | 全屏礼花、连击 `xN`、经验条上涨、角色轻跳 | 清脆成功音、连击递进音高 | 鼓励、庆祝 |
| 答错 | 卡通巴掌层、界面轻抖、红色冲击闪一下 | 噼里啪啦/巴掌声 | 搞怪不羞辱 |
| 复仇成功 | 特殊金色翻盘礼花、卡片从红变绿动画 | 胜利+「翻盘」音效 | 「这笔债，你讨回来了」 |

**设置项：**

- 音效总开关、音量  
- 巴掌强度：关 / 轻 / 狂  
- 礼花强度：关 / 标准 / 全开  
- 动效减弱（无障碍/性能）  

### 4.5 今日副本（每日练习）

时长目标：**8～12 分钟**，结构固定：

1. **热身 2 题** — 昨天会的或更低一档，建立信心  
2. **主线 5 题** — 当前地图难度  
3. **挑战 1 题** — 略难一档，成功双倍经验  
4. **复仇 1～2 题** — 从待翻盘优先抽取  

完成后：

- 日历 **今日印章**  
- 经验 / 金币（或等价「星尘」）  
- 一句个性化鼓励（基于当日数据模板，非空话）  
- 连续打卡火焰：3 / 7 / 21 天称号；断签火焰减弱，可用「补签小任务」挽回（第一版可做简化：仅展示连续天数，补签为可选增强）

### 4.6 随机弹窗引擎

| 参数 | 默认 | 说明 |
|------|------|------|
| 间隔 | 30～60 分钟随机 | 可在设置中改为 15～120 分钟区间 |
| 静音时段 | 用户可配（如 12:00–13:30、23:00–08:00） | 时段内不弹 |
| 每日上限 | 8 次 | 防止厌烦 |
| 抽题配比 | 70% 当前水平 / 20% 错题复仇 / 10% 简单送分 | 保信心 |
| 题长 | 仅短题（预计 ≤40 秒） | 不含 Part2 长谈 |
| 交互 | 系统通知 + 置顶答题小窗；支持「稍后 10 分钟」 | 托盘可暂停弹题 |

通知文案示例：地图化、口语化，避免「请完成第 12 题」。

### 4.7 荣誉墙 / 复仇墙 / 待翻盘

| 系统内部概念 | 小白向名称 | 规则 |
|--------------|------------|------|
| 常胜榜 | **荣誉墙** | 记录某题/词的累计答对次数；次数↑颜色浅绿→翠绿→鎏金 |
| 耻辱榜 | **复仇墙** | 累计答错次数；次数↑粉→红→深红；一键复仇重练 |
| 错题本 | **待翻盘** | 仍处于「未征服」状态的条目；与复仇墙强关联 |

**迁移规则（明确）：**

- 任意答错：该题 `wrongCount++`，进入/留在待翻盘；更新复仇墙红度  
- 任意答对：该题 `correctCount++`；更新荣誉墙绿度  
- 当某题在「待翻盘」期间 **连续答对 2 次**（或累计答对达到阈值且最近 2 次正确）：标记为已征服，移出待翻盘；若曾在复仇墙，播放翻盘动画  
- 同一题可同时在荣誉墙展示「历史征服次数」，在复仇墙展示「历史失手次数」（双轨统计，不互相抹除）  

**筛选：** 单词 / 句子 / 话题；按次数排序。

文案基调：复仇墙 =「这些题还欠你一场胜利」，而非羞辱。

### 4.8 混合题型

| 题型 ID | 名称 | 交互 | 主要出现位置 |
|---------|------|------|----------------|
| `listen_choose` | 听音选图/选词 | 播放音频 → 点选 | 低阶地图、弹窗 |
| `word_choose` | 看义选词/看词选义 | 点选 | 低阶～中阶、弹窗 |
| `fill_blank` | 填空/选词填空 | 点选或短输入 | 中阶 |
| `reorder` | 句子排序 | 拖拽或点选顺序 | 中阶 |
| `shadow_read` | 跟读 | 示范音 → 用户说 → 识别/完整度轻量分 | 全地图、弹窗短句 |
| `short_speak` | 看题短说 | 15～45 秒录音/识别要点 | 校园港起，**不进默认弹窗** |
| `dialogue_next` | 情景下一句 | 选或说下一句 | 对话山谷起 |
| `topic_card` | 话题卡（考试感） | 准备 + 作答（简化评分） | 考场前线～雅思塔，**章节内** |

评分策略（第一版）：

- 客观题：精确匹配或选项 ID  
- 跟读：Web Speech API 文本与目标句相似度（归一化、忽略大小写标点）；给 1～3 星，**低于阈值算错但展示差异**  
- 短说/话题：关键词命中 + 时长达标的规则分；展示「你提到了…还可以加…」模板反馈；不做虚假精确 band 分

## 6. 进度、等级与经济（轻量）

### 6.1 经验与地图进度

- 每题基础经验：按题型与难度加权（简单 10、标准 15、挑战 30）  
- 连击加成：从 `x3` 起小幅加成，有上限  
- 地图进度条：当前地图完成「关卡节点」后解锁下一地图  
- 关卡节点：每个地图内 N 个关卡（建议 5～8），每关固定题包 + 随机池  

**定级与降级：**

- 连续高正确率且挑战题通过 → 建议晋级（用户确认或自动）  
- 近期正确率过低 → 地图内降档抽题，不强制踢回上一地图（保护自尊）  

### 6.2 货币与成就（第一版）

- **星尘（金币）**：完成副本、连击、成就发放；用于解锁头像框/主题色（本地装饰，无付费）  
- **勋章成就**示例：首胜、连击 10、复仇 10、点亮某地图、连续打卡 7 天、开口 100 次  
- **周报**：本周开口次数、征服词数、最长连击、最强地图、待翻盘剩余  

### 6.3 颜色强度公式（双榜）

设答对次数为 `c`，答错次数为 `w`：

- 荣誉墙绿度：`greenIntensity = min(1, log(1+c) / log(1+C_max))`，`C_max` 默认 20  
- 复仇墙红度：`redIntensity = min(1, log(1+w) / log(1+W_max))`，`W_max` 默认 10  
- UI 用强度映射到背景/边框/徽章色阶（至少 4 档可视化）

## 7. 题库设计

### 7.1 内容范围

自研 **分级示例题库**，覆盖幼儿园感 → 雅思/托福任务类型，**不声称官方真题**。

第一版体量目标（可运行、可感知进阶）：

- 每地图至少 **40** 条可用题（短题为主）  
- 全库合计 **≥ 280** 题/词条级练习单元  
- 每题含：唯一 ID、地图层级、题型、题干、答案/要点、解析或示范句、标签（单词/句子/话题）、是否允许弹窗、预估时长秒  

### 7.2 题目数据模型（逻辑）

```ts
type Question = {
  id: string;
  mapId: string;           // yayacun | wordtown | ...
  tier: 0 | 1 | 2 | 3 | 4 | 5 | 6;
  type: QuestionType;
  prompt: string;          // 题干（中英可组合）
  promptLocale?: 'zh' | 'en' | 'bilingual';
  choices?: string[];      // 选择题
  answer: string | string[]; // 标准答案或可接受列表
  keywords?: string[];     // 短说评分关键词
  sampleAnswer?: string;   // 示范回答
  audioText?: string;      // TTS/跟读目标文本
  tags: string[];          // word | sentence | topic | ...
  allowPopup: boolean;
  estSeconds: number;
  explanation?: string;
};
```

### 7.3 存储与扩展

- 题库：应用内静态 JSON（可分地图文件）  
- 用户数据：本地 `electron-store`（或等价 JSON 持久化）  
- 后续可热更新题包：预留 `contentVersion` 字段

## 9. 技术架构

### 9.1 栈

| 层 | 选型 |
|----|------|
| 壳 | Electron（Windows） |
| UI | React + TypeScript + Vite |
| 样式 | CSS Modules 或 Tailwind（实现阶段二选一，推荐 Tailwind 加速商业化 UI） |
| 动效 | CSS + Canvas 礼花；巴掌用 DOM/Canvas 动画层 |
| 音效 | Web Audio / HTMLAudioElement，资源放 `public/sounds` |
| 语音 | Web Speech API（识别 + 可选 speechSynthesis 示范） |
| 状态 | React 状态 + 持久化服务封装 |
| 存储 | electron-store（主进程或 preload 安全封装） |
| 打包 | electron-builder → Windows 安装包 / 便携版 |

### 9.2 进程职责

**主进程：**

- 创建主窗口、弹题窗  
- 托盘图标与菜单  
- 全局快捷键（可选）  
- 弹题调度器（timer，考虑休眠/静音时段）  
- 系统通知  
- 持久化读写（若放主进程）  

**预加载 preload：**

- 通过 contextBridge 暴露有限 API：`notify`、`popup`、`store`、`scheduler`  

**渲染进程：**

- 全部学习 UI、动效、语音、题库逻辑、评分展示  

### 9.3 关键模块划分

```
src/
  main/           # Electron 主进程
  preload/
  renderer/
    app/          # 路由与壳布局
    features/
      map/        # 冒险地图
      dungeon/    # 今日副本
      practice/   # 自由练习与答题器
      review/     # 待翻盘
      boards/     # 荣誉墙 / 复仇墙
      achievements/
      settings/
      onboarding/
      effects/    # 礼花、巴掌、音效控制器
    content/      # 题库 JSON
    shared/       # 类型、工具、UI 组件
```

### 9.4 安全

- `contextIsolation: true`，`nodeIntegration: false`  
- 最小权限 preload API  
- 不加载远程任意脚本（第一版纯本地）

## 10. 音效与动效资源策略

第一版可用：

- 程序化 Web Audio 生成简单成功/失败音（保证零依赖可运行）  
- 或打包少量短 wav/mp3（成功、连击、巴掌、翻盘）  
- 礼花：Canvas 粒子  
- 巴掌：全屏半透明层 + 手掌 SVG/PNG 位移动画 + 屏幕震动 CSS  

提供 **动效减弱** 时：取消粒子与强震动，仅保留色闪与 Toast。

## 11. 错误处理与边界

| 情况 | 处理 |
|------|------|
| 麦克风权限拒绝 | 语音题降级为「看示范 + 自检完成」按钮，并提示去设置开启 |
| 语音识别不可用 | 同上降级；设置中显示兼容性说明 |
| 题库某地图题不足 | 允许从相邻低阶地图借题并标注「复习」 |
| 弹题时主窗已在答题 | 跳过或合并，避免双窗抢焦点 |
| 用户重置数据 | 二次确认，不可恢复警告 |
| 通知权限失败 | 仍可用托盘定时直接弹窗 |
