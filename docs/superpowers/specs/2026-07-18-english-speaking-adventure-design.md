## 1. 背景与目标

### 1.1 要解决的问题

新手学雅思/托福口语时常见障碍：

1. **太难** — 一上来考试题，直接放弃  
2. **太枯燥** — 像写作业，没有即时反馈与成就感  
3. **没进步感** — 做了很多练习，不知道自己变强了  

### 1.2 产品定位

不是冷冰冰的「刷题软件」，而是面向**完全小白**的 **英语口语闯关成长产品**：

- 前台：地图、副本、荣誉/复仇、动效与音效  
- 后台：从幼儿园级到雅思/托福的分级题库与自适应难度  

### 1.3 成功标准（第一版「完整核心版」）

用户在第一次使用的 10 分钟内能：

1. 完成引导定级并进入合适地图  
2. 至少体验 3 次「答对 → 庆祝反馈」  
3. 完成一次「今日副本」或等价短练习  
4. 看到错题进入「待翻盘」，答对题出现在「荣誉墙」  

长期使用时能：

- 通过冒险地图从低阶地图推进到考场级地图  
- 被随机弹窗以不烦人的节奏打断并完成短题  
- 在荣誉墙/复仇墙清晰看到自己的强弱分布  

### 1.4 明确非目标（第一版不做）

- 账号体系与云同步（预留接口，第一版纯本地）  
- 付费订阅 / 内购（预留扩展点，第一版免费完整可用）  
- 真人外教 / 服务端 AI 深度评分（第一版：本地规则 + 系统语音识别）  
- iOS / Android 移动端  
- 完整官方雅思/托福真题版权内容（使用自研分级示例题库，风格对标考试任务类型）

## 2. 北极星设计原则

1. **永远让用户觉得自己会一点** — 难度自适应，整体略偏容易  
2. **每约 30 秒至少一次明确反馈** — 对/错/进度/连击  
3. **开口被奖励；错误转化为「复仇剧情」** — 不羞辱，给翻盘路径  
4. **考试目标藏在地图后面** — 前台是玩与成长，后台对齐能力分级  
5. **动效强化情绪，且可关** — 默认「完整商业化」档  
6. **弹窗只出 20～40 秒能完成的短题** — 保护「愿意开通知」的习惯

## 3. 用户与使用场景

### 3.1 核心用户

- 英语口语基础弱、想最终冲雅思/托福的成人或学生  
- 能接受「游戏化」与轻度搞怪反馈  
- 主要在 Windows 电脑前学习/摸鱼间隙练习  

### 3.2 关键场景

| 场景 | 行为 |
|------|------|
| 首次打开 | 引导 → 5 题热身定级 → 落入地图 → 送首枚勋章 |
| 每日主动学 | 打开「今日副本」：固定 30 题，答对才前进，可循环至通关 |
| 自由学 | 「自由练习」不限地图解锁、不限题量；「单词表」查基础词 |
| 碎片时间 | 托盘常驻，通知 + 置顶小窗随机短题 |
| 薄弱突破 | 在「待翻盘 / 复仇墙」主动点题重练 |
| 看不懂题 | 答题页一键「显示中文」提示 |
| 回顾成就 | 荣誉墙、成就页、成长速览 |

## 4. 产品体验设计

### 4.1 世界观与地图（前台）

用户角色：**口语冒险家**。从低阶地图走向考场地图。

| 地图显示名 | 能力大致对应 | 题型重心 |
|------------|--------------|----------|
| 咿呀村 | 幼儿园～小学低 | 听音选图、超短词、跟读单词 |
| 单词小镇 | 小学～初中词句 | 选词、填空、跟读短句 |
| 对话山谷 | 日常短对话 | 选下一句、情景跟读 |
| 校园港 | 校园/生活场景 | 短说 15～30 秒、情景问答 |
| 考场前线 | 四六级口语感 | 稍长回答、简单话题卡 |
| 托福城 | TOEFL 口语任务感 | 独立口语、听后说（简化版） |
| 雅思塔 | IELTS 口语 1～3 | Part1 短答 → Part2 长谈 → Part3 讨论感 |

**原则：** 用户看到的是地图与故事名；系统内部使用 `levelTier`（0～6）与题目标签对齐。

### 4.2 首次引导（Onboarding）

1. 约 30 秒欢迎：「今天不考试，只玩 3 个小游戏」  
2. **极简定级**：5 道从超简单逐渐变难的热身题（非长测）  
3. 根据正确率与最高答对难度，**宁可定低一点**，保证连赢体验  
4. 落入对应地图，发放 **第一枚勋章** + 礼花  
5. 引导打开「今日副本」或完成 3 题快速胜利  

要求：第一天内至少 **3 次**「答对 → 礼花/音效/升级感」。

### 4.3 核心成长回路

```
抽到「刚好会一点」的题
  → 混合互动（点选 / 跟读 / 短说）
  → 答对：礼花 + 音效 + 经验 + 连击
  → 答错：巴掌动效（可调强度）+ 正确答案 + 跟读/再试
  → 错题进入「待翻盘」，并提高被弹窗抽中的权重
  → 复仇成功：翻盘礼花 + 从复仇墙迁向荣誉墙
  → 连对 / 升级 / 点亮地图 → 继续动机
```

### 4.4 答题反馈（商业化动效）

| 结果 | 视觉 | 听觉 | 文案调性 |
|------|------|------|----------|
| 答对 | 全屏礼花、连击 `xN`、经验条上涨、角色轻跳 | 清脆成功音、连击递进音高 | 鼓励、庆祝 |
| 答错 | 卡通巴掌层、界面轻抖、红色冲击闪一下 | 噼里啪啦/巴掌声 | 搞怪不羞辱 |
| 复仇成功 | 特殊金色翻盘礼花、卡片从红变绿动画 | 胜利+「翻盘」音效 | 「这笔债，你讨回来了」 |

**设置项：**

- 音效总开关、音量  
- 巴掌强度：关 / 轻 / 狂  
- 礼花强度：关 / 标准 / 全开  
- 动效减弱（无障碍/性能）  

### 4.5 今日副本（每日练习）— 现行规则 v2

**固定 30 题**，结构：

| 阶段 | 题量 | 说明 |
|------|------|------|
| 热身 | 5 | 偏低一档或当前档，建立信心 |
| 主线 | 15 | 当前地图难度 |
| 挑战 | 5 | 略高一档 |
| 复仇 | 5 | 优先待翻盘；不足则从题库补齐 |

**通关规则（循环挑战）：**

1. **只有答对才能进入下一题**  
2. **答错不前进**：原地再战同一题，可无限重试，并统计「本题已错 N 次」  
3. 30 题全部答对 → 通关、盖章、发星尘、解锁「副本启程」类成就  
4. 通关后或中途可点 **「重开 30 题」** 再开一轮（重新抽题）  
5. 旧版短副本（&lt;30 题）打开时自动迁移为 30 题  

完成后：

- 经验 / 星尘  
- 连续打卡天数累计  
- 可再开一轮 30 题  

### 4.5.1 自由练习（现行）

- **不设 12 题上限**；按筛选条件使用全量题库  
- **不要求地图解锁**：全部地图可选  
- 支持：全部地图 / 单地图、全部题型 / 单题型、上一题 / 下一题 / 重新打乱  
- 进度条展示当前筛选下的位置  

### 4.5.2 单词表（现行新增）

独立导航页 **单词表**：

- 内置 A1～B2 基础词（生成脚本写入 `vocabulary.json`，约 400+ 词）  
- 中英搜索、级别筛选  
- 释义、例句、听发音 / 听例句（TTS）  
- 与题库词条同源扩展，练习题由同一词表批量生成  

### 4.5.3 题目中文提示（现行新增）

答题器提供 **「显示中文」** 开关：

- `promptZh`：题干中文  
- `sampleAnswerZh`：示范中文  
- `explanation`：解析  
- 默认关闭，避免依赖翻译；小白可随时打开  

### 4.5.4 语音识别（现行）

- 主进程放行麦克风权限；识别前 `getUserMedia` 预检  
- Web Speech API（`en-US`），中间结果展示；跟读评分偏宽松  
- 失败路径：中文错误说明 + 打字 + 「我已跟读/说完」  
- **注意：** Chromium 语音识别通常需 **联网** + 系统麦克风权限

### 4.6 随机弹窗引擎

| 参数 | 默认 | 说明 |
|------|------|------|
| 间隔 | 30～60 分钟随机 | 可在设置中改为 15～120 分钟区间 |
| 静音时段 | 用户可配（如 12:00–13:30、23:00–08:00） | 时段内不弹 |
| 每日上限 | 8 次 | 防止厌烦 |
| 抽题配比 | 70% 当前水平 / 20% 错题复仇 / 10% 简单送分 | 保信心 |
| 题长 | 仅短题（预计 ≤40 秒） | 不含 Part2 长谈 |
| 交互 | 系统通知 + 置顶答题小窗；支持「稍后 10 分钟」 | 托盘可暂停弹题 |

通知文案示例：地图化、口语化，避免「请完成第 12 题」。

### 4.7 荣誉墙 / 复仇墙 / 待翻盘

| 系统内部概念 | 小白向名称 | 规则 |
|--------------|------------|------|
| 常胜榜 | **荣誉墙** | 记录某题/词的累计答对次数；次数↑颜色浅绿→翠绿→鎏金 |
| 耻辱榜 | **复仇墙** | 累计答错次数；次数↑粉→红→深红；一键复仇重练 |
| 错题本 | **待翻盘** | 仍处于「未征服」状态的条目；与复仇墙强关联 |

**迁移规则（明确）：**

- 任意答错：该题 `wrongCount++`，进入/留在待翻盘；更新复仇墙红度  
- 任意答对：该题 `correctCount++`；更新荣誉墙绿度  
- 当某题在「待翻盘」期间 **连续答对 2 次**（或累计答对达到阈值且最近 2 次正确）：标记为已征服，移出待翻盘；若曾在复仇墙，播放翻盘动画  
- 同一题可同时在荣誉墙展示「历史征服次数」，在复仇墙展示「历史失手次数」（双轨统计，不互相抹除）  

**筛选：** 单词 / 句子 / 话题；按次数排序。

文案基调：复仇墙 =「这些题还欠你一场胜利」，而非羞辱。

### 4.8 混合题型

| 题型 ID | 名称 | 交互 | 主要出现位置 |
|---------|------|------|----------------|
| `listen_choose` | 听音选图/选词 | 播放音频 → 点选 | 低阶地图、弹窗 |
| `word_choose` | 看义选词/看词选义 | 点选 | 低阶～中阶、弹窗 |
| `fill_blank` | 填空/选词填空 | 点选或短输入 | 中阶 |
| `reorder` | 句子排序 | 拖拽或点选顺序 | 中阶 |
| `shadow_read` | 跟读 | 示范音 → 用户说 → 识别/完整度轻量分 | 全地图、弹窗短句 |
| `short_speak` | 看题短说 | 15～45 秒录音/识别要点 | 校园港起，**不进默认弹窗** |
| `dialogue_next` | 情景下一句 | 选或说下一句 | 对话山谷起 |
| `topic_card` | 话题卡（考试感） | 准备 + 作答（简化评分） | 考场前线～雅思塔，**章节内** |

评分策略（第一版）：

- 客观题：精确匹配或选项 ID  
- 跟读：Web Speech API 文本与目标句相似度（归一化、忽略大小写标点）；给 1～3 星，**低于阈值算错但展示差异**  
- 短说/话题：关键词命中 + 时长达标的规则分；展示「你提到了…还可以加…」模板反馈；不做虚假精确 band 分

## 6. 进度、等级与经济（轻量）

### 6.1 经验与地图进度

- 每题基础经验：按题型与难度加权（简单 10、标准 15、挑战 30）  
- 连击加成：从 `x3` 起小幅加成，有上限  
- 地图进度条：当前地图完成「关卡节点」后解锁下一地图  
- 关卡节点：每个地图内 N 个关卡（建议 5～8），每关固定题包 + 随机池  

**定级与降级：**

- 连续高正确率且挑战题通过 → 建议晋级（用户确认或自动）  
- 近期正确率过低 → 地图内降档抽题，不强制踢回上一地图（保护自尊）  

### 6.2 货币与成就（第一版）

- **星尘（金币）**：完成副本、连击、成就发放；用于解锁头像框/主题色（本地装饰，无付费）  
- **勋章成就**示例：首胜、连击 10、复仇 10、点亮某地图、连续打卡 7 天、开口 100 次  
- **周报**：本周开口次数、征服词数、最长连击、最强地图、待翻盘剩余  

### 6.3 颜色强度公式（双榜）

设答对次数为 `c`，答错次数为 `w`：

- 荣誉墙绿度：`greenIntensity = min(1, log(1+c) / log(1+C_max))`，`C_max` 默认 20  
- 复仇墙红度：`redIntensity = min(1, log(1+w) / log(1+W_max))`，`W_max` 默认 10  
- UI 用强度映射到背景/边框/徽章色阶（至少 4 档可视化）

## 7. 题库设计

### 7.1 内容范围（现行）

自研 **分级示例题库**，覆盖幼儿园感 → 雅思/托福任务类型，**不声称官方真题**。

| 指标 | 初版目标 | 现行（约） |
|------|----------|------------|
| 练习题总量 | ≥ 280 | **~2013** |
| 单词表词条 | — | **~418**（A1～B2） |
| 每地图 | ≥ 40 | 均 ≥ 200（高阶由学术词补量） |

生成脚本：`scripts/generate-question-bank.mjs`  
输出：

- `src/renderer/src/content/questions.json`  
- `src/renderer/src/content/vocabulary.json`  

命令：`npm run generate:bank`

### 7.2 题目数据模型（逻辑 · 现行）

```ts
type Question = {
  id: string
  mapId: MapId
  tier: 0 | 1 | 2 | 3 | 4 | 5 | 6
  type: QuestionType
  prompt: string
  promptZh?: string          // 中文题干（显示中文开关）
  promptLocale?: 'zh' | 'en' | 'bilingual'
  choices?: string[]
  choicesZh?: string[]
  answer: string | string[]
  keywords?: string[]
  sampleAnswer?: string
  sampleAnswerZh?: string
  audioText?: string
  tags: string[]
  allowPopup: boolean
  estSeconds: number
  explanation?: string
}

type VocabEntry = {
  id: string
  word: string
  meaningZh: string
  pos?: string
  level: 'A1' | 'A2' | 'B1' | 'B2' | 'C1' | 'basic'
  mapId?: MapId
  example?: string
  exampleZh?: string
}
```

### 7.3 存储与扩展

- 题库 / 单词表：应用内静态 JSON  
- 用户数据：本地 `electron-store`（`ziqing-data`）

## 11. 错误处理与边界

| 情况 | 处理 |
|------|------|
| 麦克风权限拒绝 | 中文说明 + 打字 / 「我已跟读/说完」 |
| 语音识别需网络失败 | 提示 network，降级打字 |
| 题库某地图不足 | 从全库 / 弹窗池补齐（副本凑满 30） |
| 旧副本 &lt;30 题 | 自动重建 30 题 |
| 用户重置数据 | 二次确认 |
| 通知权限失败 | 托盘仍可直接弹窗 |
